# Setting Up MongoDB Atlas for Your Project

Your project is already configured to automatically connect to a MongoDB Atlas cluster if the correct environment variable is provided! Here is the step-by-step procedure to create your cluster and link it to your backend.

## Step 1: Create a MongoDB Atlas Account & Cluster
1. **Sign Up / Log In**: Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas/register) and register for a free account.
2. **Build a Database**:
   - Once logged in, click **Build a Database** (or **Create** under the *Database Deployments* section).
   - Choose the **M0 FREE** tier (Shared).
   - Select your preferred cloud provider (AWS, Google Cloud, or Azure) and a region closest to your location for better performance.
   - Click **Create**.

## Step 2: Configure Security Settings
While the cluster is being provisioned, MongoDB will ask you to set up security.

1. **Create a Database User**:
   - Enter a **Username** (e.g., `admin`).
   - Enter a strong **Password** (or use the autogenerated one). 
   - *Important:* If your password contains special characters like `@`, you will need to URL-encode them in the connection string later (e.g., `@` becomes `%40`).
   - Click **Create User**.
2. **Whitelist Your IP Address**:
   - In the "Network Access" step, click **Add My Current IP Address** to allow your local machine to connect.
   - *Alternative:* If you want to connect from anywhere (not recommended for production, but fine for testing), click **Allow Access from Anywhere** (which inserts `0.0.0.0/0`).
   - Click **Finish and Close**.

## Step 3: Get Your Connection String
1. Go back to the **Database Deployments** page.
2. Click the **Connect** button next to your new cluster.
3. Select **Drivers** (Connect to your application).
4. **Copy the Connection String** provided. It will look something like this:
   ```text
   mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0
   ```
5. Replace `<password>` with the password you created in Step 2. (Remember to URL-encode special characters if necessary).

## Step 4: Configure Your Local Environment
The `backend/src/server.js` file is already designed to look for an environment variable named `MONGO_URI`. 

1. Create a file named `.env` in the `backend/` directory of your project (i.e., `/Users/maheshvellogi/Downloads/Final FSD/FSD_project/backend/.env`).
2. Add your Atlas connection string to this file:
   ```env
   # backend/.env
   PORT=5001
   MONGO_URI="mongodb+srv://<your_username>:<your_password>@cluster0.xxxxx.mongodb.net/smartedu?retryWrites=true&w=majority&appName=Cluster0"
   ```
   *(Note: Adding `/smartedu` before the `?` query parameters sets the name of the database that will be created inside your cluster.)*

## Step 5: Test the Connection
1. Stop your currently running backend server in the terminal (press `Ctrl + C`).
2. Make sure the `.env` file is saved in the `backend/` folder.
3. Restart the backend:
   ```bash
   cd FSD_project/backend
   npm start
   ```
4. Look at the terminal output. You should see:
   ```
   Connected to MongoDB
   Backend API listening on http://localhost:5001
   ```
   Your application is now successfully using MongoDB Atlas in the cloud!
